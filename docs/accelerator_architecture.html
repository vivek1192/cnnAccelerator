③ 8×8 PE Array — Spatial Layout
Each row processes one input channel (ic). Each column processes one output channel (oc). Weights are broadcast ▼ down each column (same weight to all 8 PEs in a column). Activations are broadcast ► across each row (same activation to all 8 PEs in a row). Partial sums stay in the accumulator for the entire input-channel tile — never written off-chip mid-tile.
▼ Weight broadcast (per column, per cycle) — 4-bit signed ► Activation broadcast (per row, per cycle) — 8-bit unsigned ● Accumulator drains after full tile — 24-bit → quantize_scale

Tile iteration order (outermost → innermost):
for oc_tile → for oh_tile → for ow_tile → for ic_tile → COMPUTE [all 64 PEs] → DRAIN
After all ic_tiles complete: drain 64 accumulators → quantize_scale → relu → output SRAM. Then move to next ow/oh/oc tile.
④ Single PE — Cycle-Accurate Internal Pipeline
Each PE has a 2-stage Booth multiplier feeding a saturating accumulator. The accumulator runs every cycle, accumulating one MAC per cycle after the initial 2-cycle pipeline fill. The accumulator clears synchronously at tile boundaries driven by the Layer Controller.

⑤ Booth Multiplier — Radix-2, 2-Stage Pipelined
The Booth multiplier computes signed 4-bit weight × unsigned 8-bit activation → 12-bit product in 2 clock cycles. Radix-2 Booth recoding processes 1 bit per step, generating 2 partial products for a 4-bit multiplier. In synthesis, Vivado maps this to DSP48E2 primitives.

Inputs

weight_i [3:0] ← signed, 2's complement
range: −8 to +7
activation_i [7:0] ← unsigned
range: 0 to 255
Max product magnitude:
|−8| × 255 = 2040 → fits in 12 bits (max 2^11 = 2048) ✓

Stage 1 — Partial Product Generation (Cycle N)

// Booth recoding: process pairs with overlap
// Extended multiplier: {w[3], w[3:0], 1'b0}
pair0 = {w[1], w[0], 1'b0}
pair1 = {w[3], w[2], w[1]}
// Each pair encodes: −2, −1, 0, +1, +2
pp0 = encode(pair0) × activation // shifted 0
pp1 = encode(pair1) × activation // shifted 2
// Register partial products
pp0_r [11:0] ← registered
pp1_r [11:0] ← registered (shifted 2)

Stage 2 — Wallace Tree Reduction (Cycle N+1)

// Sum two 12-bit partial products
product_o [11:0] = pp0_r + pp1_r
// 12-bit adder, no overflow
// (proven by input range analysis)
// Assert output valid
valid_o ← 1
// valid_o = $past($past(valid_i))
// exactly 2-cycle pipeline delay
Output is registered — stable for 1 full cycle.
Accumulator samples on next posedge clk.

Output → Accumulator

// Sign-extend to 24-bit before adding
product_24 = {{12{product_o[11]}},
product_o[11:0]}
// Saturating accumulate
sum = acc_r + product_24
// Overflow detect:
if (!acc_r[23] && !product_24[23]
&& sum[23]) // pos + pos = neg?
acc_r ← 24'h7FFFFF // saturate
else
acc_r ← sum
DSP48E2 note: Vivado infers a DSP for the multiply
and a separate LUT chain for saturation mux.

⑥ Memory Subsystem — SRAM Banks & Double Buffering
Three on-chip SRAM banks are used. Double-buffering on the activation SRAM overlaps DMA prefetch of the next tile with PE computation on the current tile, hiding DDR4 access latency.

After the PE array completes a tile, all 64 accumulators drain through this pipeline. The quantize_scale unit applies layer-specific scale and shift parameters derived from QAT calibration. The rounding mode must exactly match the Python golden model.

⑧ AXI4 Interface Specifications
Three AXI4 interfaces connect the accelerator to the ARM PS. AXI4-Lite carries control register reads/writes. The AXI4 HP port carries high-bandwidth DMA bursts for weights and activations. AXI4-Stream (optional) carries streaming pixel data directly from sensor.
