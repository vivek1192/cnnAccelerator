System Overview — ZCU104 SoC Partition
The design is split across the Processing System (PS) and Programmable Logic (PL) of the Xilinx Zynq UltraScale+ ZCU104. The ARM Cortex-A53 handles data management and orchestration; the PL fabric runs the accelerator.

Accelerator Top-Level — Module Hierarchy
The accelerator is composed of six major subsystems. The Layer Controller FSM orchestrates all operations. The DMA Engine fetches data from DDR4. The PE Array computes MAC operations. Three SRAM banks hold weights, activations, and outputs.

Processing Element (PE) Internal Architecture
Each of the 64 PEs contains a local register file, a 2-stage pipelined Booth multiplier, and a 24-bit saturating accumulator. The accumulator clamps to max positive value on overflow — never wraps — critical for medical safety.

8×8 PE Array — Output-Stationary Dataflow
In output-stationary mode, partial sums stay in PE accumulators for the entire input-channel tile iteration. Weights are broadcast down each column (shared by 8 PEs with same output channel). Activations are broadcast across each row (shared by 8 PEs with same input channel).

Memory Hierarchy — Three Levels
Data moves through three levels: PE register files (fastest, smallest), on-chip SRAM banks (medium), and off-chip DDR4 (largest, slowest). The design minimises DDR4 accesses using double-buffering and output-stationary dataflow.

Layer Controller FSM — State Machine
The Layer Controller FSM orchestrates all operations: DMA requests, PE array control, tile iteration, and layer sequencing. One-hot state encoding is used for timing robustness on FPGA.

Post-Accumulation Pipeline — Quantize → ReLU → MaxPool
After each output tile is complete, all 64 PE accumulators are drained through this fixed pipeline. The quantize_scale unit converts 24-bit fixed-point to 8-bit using layer-specific scale and shift parameters that match QAT training.

Software Model — MobileNetV2 Architecture (W4A8 Quantized)
MobileNetV2 is the software model trained on NIH ChestX-ray14 using Quantization-Aware Training (QAT). The FPGA accelerator executes each layer sequentially, layer by layer, configured via AXI4-Lite registers.
